# Desk_Lyrics PRD（产品需求文档）

## 1. 项目概述

### 1.1 项目名称
Desk_Lyrics — 局域网实时歌词投射器

### 1.2 项目愿景
让用户在 PC 端使用网易云音乐听歌时，局域网内的其他设备（如平板）只需打开一个网页，即可实时看到同步滚动的歌词，享受沉浸式的歌词观赏体验。

### 1.3 核心价值
- **零配置体验**：平板只需打开网页，无需安装任何 App
- **实时同步**：歌词与 PC 端播放进度精确同步
- **视觉享受**：多种精美主题，让歌词展示成为一种美学体验
- **轻量部署**：一条命令启动，即开即用

---

## 2. 目标用户与场景

### 2.1 目标用户
- 在 PC 上使用网易云音乐听歌的用户
- 拥有平板或其他局域网设备作为第二屏幕

### 2.2 典型使用场景
1. **桌面歌词屏**：PC 放歌，平板放在桌面上作为专属歌词显示器
2. **远距离观看**：PC 连接音响，坐在稍远的位置通过平板看歌词
3. **分享歌词**：与朋友一起听歌时，将歌词投射到大屏设备上

## 3. 功能需求

### 3.1 核心功能（P0）

#### 3.1.1 实时播放状态获取
- 自动发现并连接网易云音乐 PC 客户端的本地 HTTP API
- 实时获取当前播放歌曲信息（歌曲 ID、名称、艺术家、专辑、封面）
- 实时获取播放进度（当前时间、总时长、播放/暂停状态）
- 播放状态变化时自动推送更新（切歌、暂停/播放）

#### 3.1.2 歌词获取与解析
- 根据歌曲 ID 从网易云云 API 获取歌词
- 解析 LRC 格式歌词，提取时间戳和文本
- 支持获取翻译歌词，并与原文按时间戳匹配
- 歌词缓存机制，避免重复请求

#### 3.1.3 实时歌词同步显示
- 根据播放进度高亮当前歌词行
- 平滑滚动到当前歌词行
- 歌曲切换时自动加载新歌词
- 暂停时歌词停止滚动，恢复播放后继续同步

#### 3.1.4 局域网 Web 服务
- 启动本地 HTTP + WebSocket 服务器
- 自动获取本机局域网 IP 地址
- 在控制台显示可访问的 URL（含二维码）
- 支持多设备同时连接

### 3.2 重要功能（P1）

#### 3.2.1 多主题系统
提供至少 3 种视觉主题，用户可在页面内切换：

| 主题 | 风格描述 | 关键视觉元素 |
|------|---------|-------------|
| 暗黑沉浸（Dark Immersive） | 深色背景，演唱会歌词屏感 | 纯黑/深灰背景、当前行发光放大、上下行渐隐 |
| 野兽主义（Brutalism） | 粗犷、高对比、反常规 | 鲜艳撞色、粗边框、大号无衬线字体、非常规布局 |
| Apple Music 风 | 优雅、流畅、现代 | 渐变背景（取自专辑封面主色）、毛玻璃效果、流畅滚动动画 |

- 主题切换时平滑过渡
- 主题选择持久化（localStorage）

#### 3.2.2 基础播放控制
平板端提供以下控制按钮：
- 播放 / 暂停
- 上一首 / 下一首
- 控制操作通过后端转发到网易云客户端

#### 3.2.3 歌曲信息展示
- 显示当前歌曲名称、艺术家、专辑名
- 显示专辑封面图片
- 封面图片作为 Apple Music 主题的渐变色来源

### 3.3 加分功能（P2）

#### 3.3.1 连接状态指示
- 显示与后端的 WebSocket 连接状态
- 断线自动重连
- 网易云客户端未运行时的友好提示

#### 3.3.2 歌词手动偏移校准
- 允许用户微调歌词时间偏移（±5s）
- 针对歌词与音频不同步的情况

#### 3.3.3 全屏模式
- 一键进入浏览器全屏，隐藏所有控制元素
- 适合将平板作为纯歌词屏使用

#### 3.3.4 字体大小调节
- 适应不同观看距离的需求
- 设置持久化

### 4.4 数据流设计

#### 4.4.1 播放状态获取流程
```
后端定时器 (500ms)
    │
    ▼
请求网易云本地 API ──▶ 获取当前歌曲 ID + 播放进度
    │
    ├── 歌曲变化？ ──▶ 请求云 API 获取歌词 ──▶ 解析+缓存 ──▶ 推送完整数据
    │
    └── 仅进度变化？ ──▶ 推送进度更新
```

#### 4.4.2 WebSocket 消息协议

**服务端 → 客户端：**

| 消息类型 | 字段 | 说明 |
|---------|------|------|
| `song:change` | `{ songId, name, artist, album, coverUrl, duration }` | 歌曲切换时推送 |
| `lyrics:load` | `{ original: [{time, text}], translation: [{time, text}] }` | 歌词加载完成 |
| `progress:update` | `{ currentTime, duration, isPlaying }` | 播放进度更新（~500ms） |
| `player:state` | `{ isPlaying }` | 播放/暂停状态变化 |

**客户端 → 服务端：**

| 消息类型 | 字段 | 说明 |
|---------|------|------|
| `control:play` | `{}` | 播放 |
| `control:pause` | `{}` | 暂停 |
| `control:prev` | `{}` | 上一首 |
| `control:next` | `{}` | 下一首 |
| `theme:change` | `{ theme }` | 主题切换通知（可选，用于服务端日志） |

### 4.5 网易云本地 API 对接方案

#### 4.5.1 API 端口发现
网易云音乐 PC 客户端启动时会在本地开启 HTTP 服务。端口发现策略：
1. 读取客户端配置文件获取端口号（路径：`%USERPROFILE%\AppData\Local\Netease\CloudMusic\` 下的相关配置）
2. 若配置文件不可用，扫描常用端口范围（如 3000-65535）
3. 通过发送测试请求验证端口有效性

#### 4.5.2 核心 API 端点（基于社区逆向）
- `GET /api/player/status` — 获取播放状态
- `GET /api/player/current` — 获取当前歌曲信息
- `GET /api/player/progress` — 获取播放进度
- `POST /api/player/play` — 播放
- `POST /api/player/pause` — 暂停
- `POST /api/player/next` — 下一首
- `POST /api/player/prev` — 上一首

> ⚠️ 注意：以上端点基于社区逆向工程，可能随客户端版本更新而变化。实现时需做好兼容性处理和降级方案。

### 4.6 网易云云 API 对接方案

用于获取歌词和歌曲详情。核心接口：
- 歌词接口：传入歌曲 ID，返回 LRC 格式原文歌词 + 翻译歌词
- 歌曲详情接口：传入歌曲 ID，返回歌曲名称、艺术家、专辑、封面等

实现方式：在项目中集成网易云 API 的加密逻辑（参考 NeteaseCloudMusicApi 开源项目的加密实现），直接调用云 API，无需额外启动服务。

## 5. 前端设计规范

### 5.1 主题详细设计

#### 5.1.1 暗黑沉浸风（Dark Immersive）
- **背景**：纯黑 (#000000) 或极深灰 (#0a0a0a)
- **当前歌词行**：白色 (#ffffff)，字号放大至 1.5 倍，文字发光效果 (text-shadow)
- **相邻行**：灰白色，透明度渐变（距离当前行越远越淡）
- **非当前行**：深灰色，低透明度
- **动画**：当前行切换时平滑缩放 + 淡入淡出
- **整体氛围**：演唱会歌词屏、KTV 歌词感

#### 5.1.2 野兽主义（Brutalism）
- **背景**：纯白或高饱和度单色色块
- **文字**：黑色无衬线粗体，极大字号
- **边框**：粗黑边框 (3-4px)，元素之间硬边界
- **配色**：红/黄/蓝原色撞色，高对比
- **布局**：非常规，歌词可能倾斜、重叠、错位排列
- **动画**：生硬切换，无过渡，或使用突兀的位移
- **整体氛围**：反设计、粗犷、有冲击力

#### 5.1.3 Apple Music 风
- **背景**：从专辑封面提取主色调生成的渐变背景
- **文字**：SF Pro 风格字体，白色，柔和阴影
- **当前歌词行**：加粗 + 轻微放大，毛玻璃背景条
- **动画**：流畅的弹性滚动，当前行居中
- **封面**：顶部显示专辑封面（圆角、阴影）
- **整体氛围**：优雅、精致、现代

### 5.2 响应式设计
- 适配平板横屏/竖屏
- 适配手机屏幕（备用场景）
- 字体大小随屏幕宽度自适应

### 5.3 歌词渲染逻辑
1. 歌词容器居中显示，当前行始终保持在视口中央
2. 当前行与相邻行之间有明显的视觉层级差异
3. 滚动使用 CSS `scroll-behavior: smooth` 或 JS 平滑滚动
4. 双语歌词：原文在上，翻译在下（字号略小、颜色略淡）

## 6. 非功能需求

### 6.1 性能
- 歌词同步延迟 < 200ms（从播放进度变化到界面更新）
- WebSocket 消息频率 ~2Hz（每 500ms 一次进度更新）
- 歌词切换动画帧率 ≥ 30fps

### 6.2 可靠性
- WebSocket 断线自动重连（指数退避策略）
- 网易云客户端未运行时显示友好提示，不崩溃
- 歌词获取失败时显示"暂无歌词"提示

### 6.3 易用性
- 启动命令：`npm start`
- 启动后自动在控制台显示访问 URL 和二维码
- 平板扫码或输入 URL 即可访问
- 零配置，开箱即用

### 6.4 兼容性
- 后端：Windows 10/11（网易云音乐 PC 客户端平台）
- 前端：iOS Safari、Android Chrome、桌面 Chrome/Edge/Firefox

## 8. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 网易云本地 API 端点变更 | 无法获取播放状态 | 做好版本检测，提供手动输入端口的降级方案 |
| 网易云云 API 加密方式变更 | 无法获取歌词 | 参考社区最新实现及时更新；本地缓存已获取的歌词 |
| 歌词与音频不同步 | 用户体验差 | 提供手动偏移校准功能 |
| 局域网防火墙阻断 | 平板无法访问 | 在启动时提示用户检查防火墙设置 |
| WebSocket 频繁断连 | 歌词卡顿 | 实现指数退避重连 + 本地进度预测 |

## 9. 成功指标

1. **功能完整性**：PC 放歌 → 平板实时显示同步歌词，端到端链路畅通
2. **同步精度**：歌词高亮与音频偏差 < 500ms
3. **视觉体验**：3 种主题均可正常切换使用，动画流畅
4. **易用性**：从启动到看到歌词，操作步骤 ≤ 3 步
5. **稳定性**：连续播放 1 小时无崩溃、无断连
